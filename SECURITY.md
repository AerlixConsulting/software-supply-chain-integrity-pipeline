# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please follow the guidelines below to report it responsibly:

- Do not publicly disclose the issue until it has been addressed.
- Email a detailed report to security@aerlixconsulting.com or open a private security advisory via GitHub.
- Provide a clear description of the vulnerability, steps to reproduce, and any potential impacts.
- We will acknowledge your report within a reasonable timeframe and work with you to fix the issue.

## Supported Versions

We maintain the `main` branch of this project. Security fixes will be applied to the latest release.

## Security Practices

This repository includes:

- Environment-based configuration to avoid committing secrets.
- Dependabot and security scanning enabled for dependency monitoring.
- Code scanning via GitHub to detect potential issues.
- Audit logging and least privilege access in deployment configurations.

Refer to the `README.md` and `docs/` for more details on secure deployment and operations.
