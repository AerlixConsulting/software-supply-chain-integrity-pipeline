# Contributing

Thank you for your interest in contributing! We welcome bug reports, enhancements, feature requests, and documentation improvements.

## How to Contribute

1. Fork this repository and clone your fork.
2. Create a feature branch (`git checkout -b feature/your-feature-name`).
3. Make your changes, following the existing code style and guidelines.
4. Ensure tests and linting pass (`pytest` or other tools).
5. Commit your changes with descriptive commit messages.
6. Push your branch to your fork (`git push origin feature/your-feature-name`).
7. Open a pull request against the `main` branch.

Please ensure that your pull request:

- Includes tests for new features or bug fixes.
- Updates documentation where appropriate.
- References any related issues in the body.
